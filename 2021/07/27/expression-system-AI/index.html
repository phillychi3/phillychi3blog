# 人臉表情辨識系統 (盲人社交輔助系統)

這是一個悲慘的專題
原本我們使用的是
微軟的 azure 來做表情辨識的部分
但是在成果發表前 14 天
指導老師表示對此不滿意
要求我們改用自己訓練的方式
神奇的老師

原項目:https://github.com/facetrach-nihs0833/expression_system
趕工後的 AI:https://github.com/facetrach-nihs0833/expression_system_AI
這是我第一次寫 AI 這種東西

一開始我們使用的是 azure 原本以為使用外部的資源會一路順暢 結果還是遇到了滿滿的困難
比如檢測線程把攝影機線程給卡住導致畫面顯示不順暢 或者是上傳過慢導致整體速度拖慢
雖然我這裡說遇到很多困難 但是我們還是比別組早完成了 5 個多月
就在別組在水深火熱之中時 我們只要寫寫報告 弄弄小論文與海報就好
然而最後報應還是來了

我們臉部辨識的部分採用了目前最知名的 opencv haarcascade_detect
而套件使用了 pytorch，單純不喜歡 tensorflow
在最初抓取了大約 5 萬張圖片進行訓練， 非常不易外的出現了過擬合
這時候有 1. 照片過少 2. 單一表情數據過多 3. 隱藏層過多 的可能 (吧
最初我們以為是 (1) 所以把照片數量擴充到 15 萬張
但依然不見任何起色
於是把單一種類照片限制為 1 萬張
這時總算辨識結果出現點不一樣的東西了

這次最主要的難點是 dataset 的部分
在很多教學網站上你會看到大家都建議你下載設定好的 pytorch dataset
但這東西只適用於比較常見的 AI 例如手寫數字辨識
反而自己訓練 dataset 成為比較稀有的東西 (?
詳細的程式我在這裡就不說了 離我寫出這東西已經過了有一段時間了 我也怕我說錯
你可以在我的專案下方查看 create_dataser.py 這個檔案
裡面有我怎麼設定 dataset 的 code

這次的專題雖然名稱為 盲人社交輔助系統 但最後主要設計的還是表情辨識的部分 有點可惜
不然可以讓這次的專題更有意義 也可以讓自己在學習歷程上面多一點可以寫的東西
很多專題在最後會由於技術問題與時間壓力讓自己偏離了遠有的想法，就像原本我想做天眼 (後來被打槍
